<objective>
Fix the dual FloatingActionButton collision in HomeScreen by suppressing the outer command-palette FAB on Cron (tab 1) and Dotfiles (tab 2) tabs, so each tab shows exactly one tappable FAB.

Purpose: The outer HomeScreen FAB (Icons.search) renders on top of the inner CronScreen/DotfilesScreen FAB (Icons.add) at the same endFloat position because IndexedStack keeps all child Scaffolds alive. This makes the inner add FAB untappable. Returning null from the outer FAB on those tabs resolves the collision.

Output: Modified `home_screen.dart` with a null branch in the floatingActionButton ternary for _kTabCron and _kTabDotfiles.
</objective>
