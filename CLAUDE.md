## What This Project Is

**AgentShell** — a Flutter app (Android + Web) that connects over WebSocket to a Rust backend. The backend manages TMUX sessions, terminal I/O, chat history (SQLite), cron jobs, dotfiles, and bridges to an AI agent (ACP/OpenCode). The Flutter app provides a terminal UI, chat interface, and session manager.

## Build Commands

All Flutter builds run inside Docker (requires `agentshell-flutter-base:latest` image).

```bash
# Flutter
cd flutter
./build.sh web               # Web build → agentshell-web.zip
./build.sh release           # Release APK
./build.sh debug             # Debug APK
./build.sh release --install # Build + install via USB
./build.sh release --wireless # Build + install via WiFi

# Backend (Rust)
cd backend-rust
cargo build --release        # → target/release/agentshell-backend
cargo run --release          # Run locally (port 4010)
cargo test
cargo bench --bench performance

# Flutter tests
cd flutter && flutter test
```

## Deploy Web

```bash
./scripts/deploy-web.sh          # Deploy existing agentshell-web.zip to S3 + purge Cloudflare
./scripts/deploy-web.sh --build  # Rebuild first, then deploy
```

Config is read from `.env` (`S3_WEB_BUCKET`, `CF_API_KEY`, `CF_EMAIL`, `CF_ZONE_ID`, `CF_DOMAIN`).

## Service Management

```bash
./start.sh start|stop|restart|logs|status
```

## Architecture

### Flutter App

State management: **Riverpod** (`StateNotifier` + `Provider`).

The central connection point is `sharedWebSocketServiceProvider` (in `flutter/lib/features/sessions/providers/sessions_provider.dart`). It creates one `WebSocketService` instance and reconnects whenever `selectedHost` changes. All features (`chat`, `terminal`, `cron`, `dotfiles`, `system`) watch this provider to get the shared WebSocket.

Feature layout follows `feature/` → `providers/` + `screens/` + `widgets/`. Data layer is `data/models/`, `data/services/`, `data/repositories/`.

Key services:
- `websocket_service.dart` — WebSocket client, auto-reconnect (5s), ping-pong keep-alive (30s)
- `terminal_service.dart` — PTY I/O bridge
- `whisper_service.dart` — Speech-to-text

**Host connection logic** (`data/models/host.dart`): port 443 or web → `wss://`/`https://`; otherwise `ws://`/`http://`. This means adding any domain on port 443 works on both web and Android automatically (Cloudflare-proxied backends included).

**Build-time config** (`core/config/build_config.dart`) is auto-generated by the Dockerfile from `.env` values (`SERVER_LIST`, `OPENAI_API_KEY`, `SHOW_THINKING`, `SHOW_TOOL_CALLS`, `BUILD_TIMESTAMP`). The `BUILD_TIMESTAMP` changes on every build to bust the Flutter service worker cache hash.

### Rust Backend

Entry point: `backend-rust/src/main.rs` sets up Axum routes and starts Tokio runtime.

`backend-rust/src/websocket/mod.rs` (~2800 lines) is the core — handles all incoming WebSocket messages from clients and routes them to subsystems (TMUX, ACP, chat, audio, dotfiles, cron).

Key subsystems:
- `tmux/` — wraps the `tmux` CLI to create/attach/kill sessions and windows
- `acp/` — ACP client that talks to the OpenCode AI agent service
- `chat_event_store.rs` — SQLite-backed chat history
- `monitor/` — watches for TMUX state changes and broadcasts updates to clients

### WebSocket Protocol

All messages are JSON. Client → server uses `type` field (e.g. `list-sessions`, `attach-session`, `input`, `resize`, `acp-create-session`). Server → client uses `type` field (e.g. `sessions-list`, `output`, `system-stats`, `acp-session-created`).

### Docker Build Flow

1. `docker/flutter-base/Dockerfile` — heavy base image (Ubuntu + Java + Android SDK + Flutter). Pushed to GHCR, cached.
2. `docker/flutter/Dockerfile` — extends base, copies source, generates `build_config.dart`, runs `flutter build web|apk`.
3. `flutter/build.sh` passes `BUILD_TIMESTAMP=$(date -u +%Y%m%dT%H%M%SZ)` so every build changes the compiled output.

### CI/CD

- `.github/workflows/flutter.yml` — builds base image, then APK and web in parallel. Uploads artifacts; attaches to GitHub release on `release` event.
- `.github/workflows/backend.yml` — `cargo build --release`, uploads Linux x86_64 binary.

## Environment (.env)

```
SERVER_LIST=host:port,Label|host:port,Label   # Embedded in APK at build time
OPENAI_API_KEY=sk-...
SHOW_THINKING=false

S3_WEB_BUCKET=s3://your-bucket

CF_EMAIL=...
CF_ZONE_ID=...
```

`.env` is gitignored. `SERVER_LIST` format: pipe-separated `address:port,Name` entries.
